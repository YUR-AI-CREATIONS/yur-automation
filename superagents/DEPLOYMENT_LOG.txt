LLM CONFIGURATION RATIONALE:

  Parameter: Temperature = 0.2
  - Scientific Rationale:
    * Temperature controls randomness in token selection
    * 0.0 = Deterministic (always select highest probability token)
    * 1.0 = High randomness (uniform distribution across possibilities)
    * 0.2 = Slight randomness with strong factuality bias
  - Why 0.2 for code analysis:
    * Code context must be interpreted literally
    * Factual accuracy > creative generation
    * Prevents hallucination of nonexistent functions/variables
    * Still allows natural language phrasing
  - Lower values if hallucination observed:
    * 0.1 = Most deterministic
    * 0.05 = Nearly mechanical (highest factuality)

  Parameter: Context Window = 4096 tokens
  - Ollama default limit: 2048 tokens
  - Our setting: 4096 tokens (covers most code analysis queries)
  - Why increased:
    * Code context requires multiple artifacts
    * System prompt + user query + 3 artifacts ≈ 2000-3000 tokens
    * Buffer for token counting accuracy
  - Trade-off: Memory usage ↑, latency ↑ (minimal at local)

  Parameter: Model = llama3
  - Size: 8B parameters
  - Advantages:
    * Good speed/quality balance for local inference
    * Trained on code-heavy corpus
    * Instruction-following capability
    * Fits on consumer hardware (≥8GB RAM, ≥4GB VRAM)
  - Alternatives:
    * mistral-7b (faster, less capable)
    * dolphin-mixtral-45b (more capable, needs 30GB+ RAM)
